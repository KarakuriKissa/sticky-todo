# 自動アップデート と コード署名の現状

最終更新: 2026-05-07

## 自動アップデート（現状）
本アプリは、起動時に GitHub Actions API へ問い合わせ、より新しい成功ビルドが見つかれば
**画面右下にバナー**で通知します。バナーから「ダウンロードページを開く」を押すと
GitHub Releases ページが既定ブラウザで開きます。

ユーザー操作:
1. バナーが出る
2. 「ダウンロードページを開く」をクリック
3. 最新の `StickyTodo_x.y.z_x64-setup.exe`（インストーラー）または `sticky-todo.exe`（単体）を入手
4. 古い版を閉じて、新版をインストール / 上書き

> **完全自動化（裏でDLして再起動）にはコード署名が必須**です。下記の理由で現時点では未対応です。

## コード署名（現状: 未対応）
本アプリは現在、**未署名（コード署名証明書を保有していない）** です。

そのため:
- 初回起動時に Windows SmartScreen が「**WindowsによってPCが保護されました**」と警告
- ユーザーは「**詳細情報** → **実行**」を一度だけクリックする必要があります
- 単体 EXE 版を Windows プロパティ画面で開くと「ブロック解除」のチェックボックスが出る場合あり

## なぜまだ未署名なのか
コード署名証明書には費用が発生します。

| プラン | 年額 | 備考 |
|---|---|---|
| Certum Open Source | 約 4,500 円 | OSS 開発者向け、もっとも安い |
| SignPath.io OSS | 無料 | OSS のみ。要申請 |
| DigiCert / Sectigo EV | 30,000〜70,000 円 | 即日 SmartScreen 通過、商用向け |

本アプリは **MIT ライセンスの OSS** として GitHub で公開しているので、SignPath.io への
申請を計画中です。承認され次第、ワークフローに署名ステップを追加します。

## 自動アップデート完成までのロードマップ
1. ✅ ビルド済みバイナリを GitHub Releases に自動公開
2. ✅ 起動時に最新ビルド番号を確認、バナー通知
3. ⏳ SignPath.io 申請＆承認
4. ⏳ 署名鍵を GitHub Actions Secrets に登録、ビルド成果物に署名
5. ⏳ `tauri-plugin-updater` を導入、署名検証付きで自動 DL ＋ 再起動

## 利用者として今できる安全策
- ダウンロード元は GitHub Releases ページ（`https://github.com/TomTomYukkie/sticky-todo/releases`）から **のみ** にしてください。
- 怪しいサイトで配布されている同名ファイルは **絶対に実行しないで** ください。
- ビルドは GitHub Actions のログ（公開）で全コミットから検証できます。
